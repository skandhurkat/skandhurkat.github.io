Hugo Theme: Academic CV

About

A Hugo theme for this website

This is a simple Hugo theme with minimal third-party dependencies for a simple academic CV style website. Initially, I used the academic theme by gcushen, which has since changed to the new HugoBlox platform with a bazillion dependencies and requires not just Hugo, but also node.js to build. The replacement theme is intentionally simple. It showcases publications, patents, projects, blog posts, provides links to social media and academic websites like Google Scholar and ORCID. I really never want to share slides using reveal.js with a presenter view, or showcase a gallery of images on my website.

So, this is a brand new codebase, started in 2026, that strips down the theme to the bare essentials. The main features of this theme are:

  • A CV-style homepage with bio, experience, publications, patents, projects, blog posts, and contact information.

  • A navigation bar that directs you to sections on the homepage (using scrollspy) when on the homepage, or to list pages for each type when not on the homepage.

  • Client-side search using Fuse.js, from the navigation bar and from the 404 page. The 404 page automatically pre-fills the search based on the URL the user was trying to access.

  • Automatic dark or light mode selection based on the user’s OS theme preference, with a button to override.

  • Automatic sorting of experience and projects by end date (so that present experience and current projects float to the top).

  • BibTeX export for publications and patents with a modal pop-up and copy-to-clipboard functionality.

  • Blog posts have a table of contents on the right (on a desktop-sized display) with a scrollspy element to remain in sync with the reader’s position on the page. The table of contents disappears if the post does not have any headings.

  • Publications allow for the full publication text on the page.

  • Optional rendering of maths (if maths=true is set in the page front matter) using MathJax.

  • Client-side filtering of projects by status (active/inactive – depending on whether the project end date is “present”) and of publications by publication type (conference/journal/preprint/etc.) with individual pages for each type of publication (e.g., /publication_types/conference).

  • JSON-LD for SEO. The mechanism collects JSON-LD from each type of object present on the page, and appends it all in a single location at the end of the page. Objects described are Person, ScholarlyArticle, BlogPosting, OrganizationRole, EducationalOccupationalCredential, CreativeWork.

  • Google Analytics for, well, analytics, and Disqus for comments. These are gated behind a cookie consent banner and are not loaded until the user clicks “Got it!” on the cookie consent banner. Clicking “Learn more” provides a modal pop-up with links to opt-out of Google Analytics or Disqus globally. The message is customized depending on whether Google Analytics, Disqus, or both, are configured. Cookie Consent is skipped if neither are configured. User privacy on the internet is something I deeply care about.

  • A copyright footer that provides a copyright range from the year of the earliest blog post to the year the website was last rendered using Hugo.

  • RSS feeds for the website and for each section (allowing users to subscribe to just publications or just blog posts, for example).

The theme pulls in the following dependencies via CDN (requiring nothing other than Hugo to build the website):

  • Bootstrap.

  • Font Awesome and Academicons for icons.

  • MathJax for rendering maths.

  • Fuse.js for search.

  • Osano Cookie Consent for, well, cookie consent.

I plan to keep this code up to date as and when I update my website with new blog posts. Feel free to create a pull request or to raise an issue.
